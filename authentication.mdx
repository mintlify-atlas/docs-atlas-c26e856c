## Overview

Tykode uses Better-Auth for secure authentication with email and password. Your account data is stored in a PostgreSQL database managed by Supabase, with industry-standard security practices.

<Note>
  All authentication endpoints use HTTPS encryption, and passwords are hashed using secure algorithms before storage.
</Note>

## Authentication architecture

The authentication system is built with:

- **Better-Auth**: Modern authentication library for Next.js
- **Prisma**: Type-safe database ORM
- **PostgreSQL**: Robust relational database via Supabase
- **Session management**: Secure HTTP-only cookies

<CodeGroup>
```typescript Better-Auth configuration
// apps/server/src/lib/auth.ts
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";
import prisma from "../../prisma";

export const auth = betterAuth({
  database: prismaAdapter(prisma, {
    provider: "postgresql",
  }),
  trustedOrigins: [process.env.CORS_ORIGIN || ""],
  emailAndPassword: {
    enabled: true,
  },
});
```

```typescript Database schema
// User model
model User {
  id            String    @id
  name          String
  email         String    @unique
  emailVerified Boolean
  image         String?
  createdAt     DateTime
  updatedAt     DateTime
  sessions      Session[]
  accounts      Account[]
}

// Session model
model Session {
  id        String   @id
  expiresAt DateTime
  token     String   @unique
  createdAt DateTime
  updatedAt DateTime
  ipAddress String?
  userAgent String?
  userId    String
  user      User     @relation(fields: [userId], references: [id])
}
```
</CodeGroup>

## Sign up

Create a new Tykode account to access services and request quotes.

### Sign up flow

<Steps>
  <Step title="Navigate to sign up page">
    Click the "Sign up" button in the navigation header to access the registration form.
  </Step>
  
  <Step title="Enter your details">
    Provide:
    - **Name**: Your full name
    - **Email**: A valid email address (must be unique)
    - **Password**: Minimum 8 characters with uppercase, lowercase, and numbers
  </Step>
  
  <Step title="Submit the form">
    Click "Create account" to register. The system will:
    - Validate your email format
    - Check password strength
    - Verify email uniqueness
    - Create your account in the database
  </Step>
  
  <Step title="Automatic sign in">
    After successful registration, you'll be automatically signed in and redirected to your dashboard.
  </Step>
</Steps>

### Sign up API

The sign up endpoint is available at `/api/auth/signup`.

<CodeGroup>
```bash Request
curl -X POST https://tykode.com/api/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jean Dupont",
    "email": "jean@entreprise.com",
    "password": "SecurePass123"
  }'
```

```json Response (Success)
{
  "success": true,
  "user": {
    "id": "user_abc123",
    "name": "Jean Dupont",
    "email": "jean@entreprise.com",
    "emailVerified": false,
    "createdAt": "2026-03-03T10:30:00Z"
  },
  "session": {
    "token": "session_xyz789",
    "expiresAt": "2026-03-17T10:30:00Z"
  }
}
```

```json Response (Error)
{
  "error": "Email already registered",
  "code": "EMAIL_EXISTS"
}
```
</CodeGroup>

### Validation rules

<AccordionGroup>
  <Accordion title="Email validation">
    - Must be a valid email format
    - Must not already be registered
    - Maximum 255 characters
    - Common format: `user@domain.com`
  </Accordion>
  
  <Accordion title="Password requirements">
    - Minimum 8 characters
    - At least one uppercase letter
    - At least one lowercase letter
    - At least one number
    - Special characters recommended but not required
  </Accordion>
  
  <Accordion title="Name validation">
    - Minimum 2 characters
    - Maximum 100 characters
    - Can include letters, spaces, and hyphens
  </Accordion>
</AccordionGroup>

<Warning>
  **Email verification**: While email verification is supported in the database schema, it's currently not enforced. You can start using your account immediately after registration.
</Warning>

## Sign in

Access your existing Tykode account with email and password.

### Sign in flow

<Steps>
  <Step title="Navigate to sign in page">
    Click "Sign in" in the navigation header or visit `/login`.
  </Step>
  
  <Step title="Enter credentials">
    Provide your registered email and password.
  </Step>
  
  <Step title="Submit the form">
    Click "Sign in" to authenticate. The system will:
    - Verify your credentials
    - Create a new session
    - Set secure session cookies
  </Step>
  
  <Step title="Redirect to dashboard">
    On successful authentication, you'll be redirected to your dashboard.
  </Step>
</Steps>

### Sign in API

The sign in endpoint is available at `/api/auth/signin`.

<CodeGroup>
```bash Request
curl -X POST https://tykode.com/api/auth/signin \
```


```json Response (Error)

  "code": "INVALID_CREDENTIALS"

```
</CodeGroup>

### Session management

When you sign in, Better-Auth creates a session with the following properties:

- **Session token**: Unique identifier stored in HTTP-only cookie
- **Expiration**: Default 14 days from creation
- **Security**: Cookies are HTTP-only, secure, and SameSite protected
- **IP tracking**: Session records the IP address for security
- **User agent**: Device information stored for session management

<CodeGroup>
```typescript Session data structure
interface Session {
  id: string;           // Unique session ID
  token: string;        // Session token (in cookie)
  userId: string;       // Associated user ID
  expiresAt: Date;      // Expiration timestamp
  ipAddress?: string;   // Client IP address
  userAgent?: string;   // Browser/device info
  createdAt: Date;      // Session creation time
  updatedAt: Date;      // Last update time
}
```
</CodeGroup>

<Note>
  Sessions are automatically refreshed on each request, extending the expiration time. This means you stay signed in as long as you're actively using Tykode.
</Note>

## Password reset

Forgot your password? Reset it securely through email verification.

### Password reset flow

<Steps>
  <Step title="Request password reset">
    Click "Forgot password?" on the sign in page and enter your email address.
  </Step>
  
  <Step title="Receive reset email">
    Check your inbox for a password reset email containing a secure link.
    
    <Note>
      The reset link expires after 1 hour for security. If expired, request a new one.
    </Note>
  </Step>
  
  <Step title="Click the reset link">
    The link will take you to a secure page where you can create a new password.
  </Step>
  
  <Step title="Create new password">
    Enter your new password (must meet the same requirements as sign up) and confirm it.
  </Step>
  
  <Step title="Sign in with new password">
    After successful reset, you'll be redirected to sign in with your new credentials.
  </Step>
</Steps>

### Password reset API

<CodeGroup>
```bash Request reset
curl -X POST https://tykode.com/api/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "jean@entreprise.com"
  }'
```

```json Response
{
  "success": true,
  "message": "Password reset email sent"
}
```

```bash Reset password
curl -X POST https://tykode.com/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "token": "reset_token_abc123",
    "password": "NewSecurePass456"
  }'
```
</CodeGroup>

<Warning>
  For security reasons, the password reset endpoint always returns success, even if the email isn't registered. This prevents email enumeration attacks.
</Warning>

## Sign out

End your session and sign out securely.

### Sign out flow

<Steps>
  <Step title="Click sign out">
    Navigate to your profile menu and click "Sign out".
  </Step>
  
  <Step title="Session termination">
    The system will:
    - Delete your session from the database
    - Clear authentication cookies
    - Clear any cached user data
  </Step>
  
  <Step title="Redirect to home">
    You'll be redirected to the Tykode homepage.
  </Step>
</Steps>

### Sign out API

<CodeGroup>
```bash Request
curl -X POST https://tykode.com/api/auth/signout \
  -H "Cookie: session=your_session_token"
```

```json Response
{
  "success": true,
  "message": "Signed out successfully"
}
```
</CodeGroup>

## Account security

### Security features

<CardGroup cols={2}>
  <Card title="Password hashing" icon="lock">
    All passwords are hashed using bcrypt before storage - we never store plain text passwords
  </Card>
  <Card title="Session encryption" icon="shield">
    Session tokens are cryptographically secure and stored in HTTP-only cookies
  </Card>
  <Card title="HTTPS only" icon="globe">
    All authentication requests require HTTPS in production
  </Card>
  <Card title="CORS protection" icon="ban">
    Cross-origin requests are restricted to trusted domains
  </Card>
</CardGroup>

### Best practices

<AccordionGroup>
  <Accordion title="Use a strong password">
    - Minimum 8 characters, but longer is better
    - Mix uppercase, lowercase, numbers, and symbols
    - Don't reuse passwords from other sites
    - Consider using a password manager
  </Accordion>
  
  <Accordion title="Keep your email secure">
    - Use a strong password for your email account
    - Enable two-factor authentication on your email
    - Don't share password reset links
    - Check for suspicious login activity
  </Accordion>
  
  <Accordion title="Sign out on shared devices">
    - Always sign out when using public computers
    - Don't check "Remember me" on shared devices
    - Clear browser cookies after signing out
  </Accordion>
  
  <Accordion title="Monitor your account">
    - Review session activity in your account settings
    - Report suspicious activity immediately
    - Update your password regularly
  </Accordion>
</AccordionGroup>

## API reference

All authentication endpoints are available at `/api/auth/[...all]` and are handled by Better-Auth.

<CodeGroup>
```typescript API route handler
// apps/server/src/app/api/auth/[...all]/route.ts
import { auth } from "@/lib/auth";
import { toNextJsHandler } from "better-auth/next-js";

export const { GET, POST } = toNextJsHandler(auth.handler);
```
</CodeGroup>

### Available endpoints

<ResponseField name="POST /api/auth/signup" type="endpoint">
  Create a new user account
  
  **Request body:**
  - `name` (string, required)
  - `email` (string, required)
  - `password` (string, required)
  
  **Response:** User object and session token
</ResponseField>

<ResponseField name="POST /api/auth/signin" type="endpoint">
  Authenticate with email and password
  
  **Request body:**
  - `email` (string, required)
  - `password` (string, required)
  
  **Response:** User object and session token
</ResponseField>

<ResponseField name="POST /api/auth/signout" type="endpoint">
  End the current session
  
  **Request:** Session cookie required
  
  **Response:** Success confirmation
</ResponseField>

<ResponseField name="POST /api/auth/forgot-password" type="endpoint">
  Request a password reset email
  
  **Request body:**
  - `email` (string, required)
  
  **Response:** Success confirmation (always returns success)
</ResponseField>

<ResponseField name="POST /api/auth/reset-password" type="endpoint">
  Reset password with token from email
  
  **Request body:**
  - `token` (string, required)
  - `password` (string, required)
  
  **Response:** Success confirmation
</ResponseField>

<ResponseField name="GET /api/auth/session" type="endpoint">
  Get current session information
  
  **Request:** Session cookie required
  
  **Response:** User object and session details
</ResponseField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't sign in">
    **Common solutions:**
    - Verify your email and password are correct
    - Check for typos (passwords are case-sensitive)
    - Try resetting your password
    - Clear browser cookies and cache
    - Ensure JavaScript is enabled
    
    If issues persist, contact support at +594 694 45 86 55
  </Accordion>
  
  <Accordion title="Didn't receive password reset email">
    **Check:**
    - Spam/junk folder
    - Email address spelling
    - Email delivery may take up to 5 minutes
    
    **Try:**
    - Request another reset email
    - Add noreply@tykode.com to your contacts
    - Try a different email address
  </Accordion>
  
  <Accordion title="Session expired">
    Sessions expire after 14 days of inactivity or when you sign out.
    
    **Solution:** Simply sign in again with your email and password.
  </Accordion>
  
  <Accordion title="Email already registered">
    If you see this error during sign up, you already have an account.
    
    **Options:**
    - Sign in with your existing credentials
    - Reset your password if you forgot it
    - Contact support if you didn't create the account
  </Accordion>
</AccordionGroup>
