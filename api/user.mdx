## Overview

Tykode uses [Better-Auth](https://better-auth.com) for authentication and user management. User data is accessed through Better-Auth's built-in API and protected tRPC procedures.

## User Model

The User model is defined in the Prisma schema and managed by Better-Auth:

```prisma User Schema
model User {
  id            String    @id
  name          String
  email         String    @unique
  emailVerified Boolean
  image         String?
  createdAt     DateTime
  updatedAt     DateTime
  sessions      Session[]
  accounts      Account[]
}
```

<ResponseField name="name" type="string">

<ResponseField name="email" type="string">

</ResponseField>

</ResponseField>

  Array of active user sessions.
</ResponseField>

<ResponseField name="accounts" type="Account[]">
  Array of linked authentication provider accounts.

## Authentication

Tykode uses Better-Auth with email and password authentication enabled:

```typescript Better-Auth Configuration
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";

export const auth = betterAuth({
  database: prismaAdapter(prisma, {
    provider: "postgresql",
  }),
  trustedOrigins: [process.env.CORS_ORIGIN || ""],
  emailAndPassword: {
    enabled: true,
  },
});
```

## Accessing User Data

### Through Protected Procedures

User data is accessible in tRPC procedures through the context:

<CodeGroup>
```typescript Protected Procedure Example
import { trpc } from "~/lib/trpc";

// This endpoint requires authentication
const data = await trpc.privateData.query();

console.log(data.user); // User object from session
// {
//   id: "user_123",
//   name: "John Doe",
//   email: "john@example.com",
//   emailVerified: true,
//   image: "https://..."
// }
```

```typescript Custom Protected Procedure
import { protectedProcedure, router } from "../lib/trpc";

export const myRouter = router({
  getUserProfile: protectedProcedure.query(({ ctx }) => {
    // ctx.session.user contains the authenticated user
    return {
      userId: ctx.session.user.id,
      name: ctx.session.user.name,
      email: ctx.session.user.email,
    };
  }),
});
```
</CodeGroup>

### Session Context

The session context is available in all protected procedures:

<ResponseField name="ctx.session.user" type="User">
  The authenticated user object containing:
  - `id` - User ID
  - `name` - User's name
  - `email` - User's email
  - `emailVerified` - Email verification status
  - `image` - Profile image URL (optional)
</ResponseField>

## tRPC Procedures

### privateData

Example protected endpoint that returns user information.

<CodeGroup>
```typescript Usage
import { trpc } from "~/lib/trpc";

try {
  const result = await trpc.privateData.query();
  
  console.log(result.message); // "This is private"

```

## Protected Procedure Middleware

The `protectedProcedure` middleware ensures authentication:

```typescript Protected Procedure Middleware
export const protectedProcedure = t.procedure.use(({ ctx, next }) => {
  if (!ctx.session) {
    throw new TRPCError({
      code: "UNAUTHORIZED",
      message: "Authentication required",
      cause: "No session",
    });
  }
  return next({
    ctx: {
      ...ctx,
      session: ctx.session,
    },
  });
});
```

### Usage in Custom Routers

<CodeGroup>
```typescript Creating Protected Endpoint
import { protectedProcedure, router } from "../lib/trpc";
import { z } from "zod";

export const userRouter = router({
  updateProfile: protectedProcedure
    .input(z.object({
      name: z.string().min(1),
    }))
    .mutation(async ({ ctx, input }) => {
      const userId = ctx.session.user.id;
      
      // Update user profile in database
      await prisma.user.update({
        where: { id: userId },
        data: { name: input.name },
      });
      
      return { success: true };
    }),
});
```
</CodeGroup>

## Error Handling

### UNAUTHORIZED Error

Thrown when accessing protected endpoints without authentication:

```typescript Handling Auth Errors
import { TRPCClientError } from "@trpc/client";

try {
  const data = await trpc.privateData.query();
} catch (error) {
  if (error instanceof TRPCClientError) {
    if (error.data?.code === "UNAUTHORIZED") {
      // Redirect to login page
      window.location.href = "/login";
    }
  }
}
```

## Related Models

### Session

Tracks user authentication sessions:

```prisma Session Schema
model Session {
  id        String   @id
  expiresAt DateTime
  token     String   @unique
  createdAt DateTime
  updatedAt DateTime
  ipAddress String?
  userAgent String?
  userId    String
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
}
```

### Account

Links external authentication providers:

```prisma Account Schema
model Account {
  id                    String    @id
  accountId             String
  providerId            String
  userId                String
  user                  User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  accessToken           String?
  refreshToken          String?
  idToken               String?
  accessTokenExpiresAt  DateTime?
  refreshTokenExpiresAt DateTime?
  scope                 String?
  password              String?
  createdAt             DateTime
  updatedAt             DateTime
}
```

## Better-Auth Integration

For authentication operations (sign up, sign in, sign out, password reset), use Better-Auth's client:

```typescript Better-Auth Client Usage
import { auth } from "~/lib/auth";

// Sign up
await auth.signUp.email({
  email: "user@example.com",
  password: "secure-password",
  name: "John Doe",
});

// Sign in
await auth.signIn.email({
  email: "user@example.com",
  password: "secure-password",
});

// Sign out
await auth.signOut();
```

Refer to the [Better-Auth documentation](https://better-auth.com/docs) for complete authentication API reference.
